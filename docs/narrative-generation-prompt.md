# 风尘录 Chronos -- 世界史叙事数据生成 Prompt 模板

> **本文档为 AI 数据生成指令。** 将本 Prompt 连同指定的 `eraId` 发送给 AI，即可生成该时代对应的完整叙事 JSON 数据文件。一次只生成一个时代。

---

## 一、系统背景

风尘录 Chronos 是一个沉浸式历史角色扮演学习平台。用户扮演历史人物，在关键历史节点面对抉择，通过选择对话选项来推动剧情走向不同历史结局。平台覆盖世界历史的各个主要时代，面向初中（九年级）、高中（中外历史纲要下）和大学（国别史）三个学段。

**核心教育目标：** 让用户在做决策的过程中自然地学习历史知识、理解历史因果、体悟历史人物的真实处境。

**用户体验流程：**
1. 用户选择一个时代（era）
2. 进入沉浸式叙事界面，系统逐幕（act）展示场景
3. 每幕先以打字机效果展示场景描述，然后逐条淡入 NPC 对话
4. NPC 对话全部展示后，从 9 条预设对话中随机抽取 3 条作为选项呈现给用户
5. 用户选择一条后，显示"事件触发"反馈（得分 + 触发事件 + 历史注解）
6. 用户点击"下一幕"，根据选择进入对应的下一幕
7. 全部幕结束后展示总评分和各幕回顾

## 二、数据结构（JSON Schema）

### 2.1 顶层结构

```json
{
  "eraId": "sw-ancient-civilizations",
  "eraName": "古代文明的产生与发展",
  "playerRole": "古埃及法老的顾问",
  "startYear": "远古",
  "endYear": "前500年",
  "acts": [...]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `eraId` | string | 是 | 时代唯一标识，必须与下文"时代列表"中的 ID 完全一致 |
| `eraName` | string | 是 | 时代中文名称 |
| `playerRole` | string | 是 | 玩家扮演的角色描述，如"古埃及法老的顾问""拿破仑的将军"。应体现代入感 |
| `startYear` | string | 是 | 起始年份或时期描述，如"前3100年""1776年""远古" |
| `endYear` | string | 是 | 结束年份或时期描述，如"前30年""1991年""至今" |
| `acts` | array | 是 | 幕数组，详见下文 |

### 2.2 幕（Act）结构

```json
{
  "id": "act-1",
  "title": "尼罗河泛滥",
  "isKeyAct": false,
  "timeMarker": "前2560年",
  "sceneDescription": "尼罗河一年一度的泛滥带来了肥沃的淤泥，农民们开始在河谷中播种。作为法老的首席顾问，你站在河岸上，思考着今年如何组织灌溉工程。",
  "npcDialogues": [
    {
      "speaker": "大祭司",
      "text": "法老陛下，今年尼罗河的水位比去年低了两指宽，恐有旱灾之兆。",
      "avatar": "祭"
    }
  ],
  "dialogues": [
    {
      "id": "d1",
      "text": "我们应该立刻修建新的水渠，将河水引向南方的干旱农田。",
      "speaker": "我",
      "avatar": "顾",
      "score": 90,
      "triggeredEvent": "你主持修建了新的灌溉水渠，南方的农田得到了充足的水源，今年的粮食产量创了新高。",
      "historicalNote": "古埃及人依赖尼罗河定期泛滥进行农业生产。他们发展了精密的灌溉系统，包括水渠和蓄水池，以应对水位变化。前王朝时期埃及人就已经掌握了基础灌溉技术。",
      "consequence": "粮食丰收",
      "nextActId": "act-2"
    }
  ]
}
```

#### Act 字段说明

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | string | 是 | 格式: `act-N` 或 `act-Na`（N 为数字，a 为分支字母） | 幕的唯一标识 |
| `title` | string | 是 | 2-8 个中文字符 | 幕标题，简洁概括本幕事件 |
| `isKeyAct` | boolean | 是 | 整个时代中约 3-5 个为 true | 关键幕标记。关键幕在 UI 上会显示"关键幕"标签，通常对应重大历史转折点 |
| `timeMarker` | string | 是 | 年份或时期描述，如"前2560年""1789年7月14日" | 显示在场景顶部，帮助用户定位时间 |
| `sceneDescription` | string | 是 | **50-100 个中文字符** | 场景描述文本。以第三人称描述当前历史场景，融入玩家角色的处境。**用于打字机效果逐字显示，必须文笔优美、引人入胜** |
| `npcDialogues` | array | 是 | 包含 2-3 个 NPC 对话对象 | NPC 预览对话。每个 NPC 对话会在场景描述之后逐条淡入显示 |
| `dialogues` | array | 是 | **恰好 9 个对话选项** | 玩家可选对话。系统每次随机抽取 3 条展示给用户 |

#### NPC 对话对象结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `speaker` | string | 是 | NPC 角色名，应为真实历史人物，如"大祭司""亚历山大""凯撒" |
| `text` | string | 是 | NPC 说的内容，15-40 个中文字符。应体现该人物的性格和立场 |
| `avatar` | string | 是 | **单个汉字**，用于圆形头像显示。取角色名字中的一个字，如"祭""亚""凯" |

#### 玩家对话选项结构

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | string | 是 | 格式: `d1` 到 `d9` | 对话选项 ID |
| `text` | string | 是 | **15-50 个中文字符**，第一人称 | 玩家角色说的话。必须以"我"的视角表达。语气应贴合玩家角色的身份和时代背景 |
| `speaker` | string | 是 | 固定填写 `"我"` | 发言者标识 |
| `avatar` | string | 是 | **单个汉字** | 玩家头像显示字符，取自 `playerRole` 的关键字 |
| `score` | number | 是 | **60-100 的整数** | 得分。反映该选择的历史合理性：90-100 为最优历史决策；80-89 为良好决策；70-79 为一般决策；60-69 为欠佳决策。**同一幕的 9 个选项应覆盖全部分数区间** |
| `triggeredEvent` | string | 是 | **20-50 个中文字符** | 选择该对话后触发的事件叙述。以第二人称"你"描述发生了什么。要有叙事感，不是枯燥的百科条目 |
| `historicalNote` | string | 是 | **50-150 个中文字符** | 历史知识注解。**必须历史准确**，包含具体年份、人名、事件等可验证信息。这是教育核心内容 |
| `consequence` | string | 是 | **10-30 个中文字符** | 选择后果的简短概括，用于评分回顾列表 |
| `nextActId` | string | 是 | 必须指向 `acts` 数组中实际存在的某个 act 的 `id` | 分支逻辑：不同对话选项指向不同的下一幕 |

## 三、分支逻辑（Branching）详解

### 3.1 核心概念

每个时代的叙事不是线性推进的，而是一棵**决策树**。玩家的每次选择都会影响后续剧情走向。

### 3.2 分支模式

典型的分支模式如下：

```
                    act-1 (起点)
                   /      \
         (高分选择)         (低分选择)
              |                |
           act-2            act-2b
          /     \              |
   (高分选择)  (低分选择)   act-3b
        |          |          /    \
     act-3      act-3b   (高分)  (低分)
      ...        ...       |       |
                  act-4b   act-4c  act-4d
```

### 3.3 分支规则

1. **主线路径**：`act-1` → `act-2` → `act-3` → ... → `act-30`，约 30 幕构成时代主线
2. **分支路径**：`act-2b`、`act-3b`、`act-4a` 等，约 10-30 幕作为分支变体
3. **高分对话（score 85-100）**：通常指向主线下一幕（最优历史走向）
4. **中等分对话（score 70-84）**：可能指向主线或轻微变体幕（如 `act-3b`）
5. **低分对话（score 60-69）**：通常指向恶化路径幕（如 `act-2b`，表示决策导致局势恶化）
6. **关键幕分支更显著**：`isKeyAct: true` 的幕，高低分选择可能指向完全不同的后续幕序列
7. **汇合机制**：分支路径在若干幕后应汇合回主线，避免树无限膨胀。例如 `act-2b` → `act-3b` → `act-4`（回到主线）

### 3.4 nextActId 指引

- **非关键幕**：9 个对话中，约 3 个高分指向主线下一个 act；约 4 个中分指向主线下一个 act 或一个轻微变体 act；约 2 个低分指向一个恶化分支 act
- **关键幕**：9 个对话中，至少 2-3 个指向完全不同的分支 act 序列（至少持续 2-3 幕再汇合）
- **所有分支最终必须汇合**：任何分支路径都不应超过 5 幕不汇合。确保所有路径最终都能到达结局
- **不存在死胡同**：每个 act 都必须有至少 1 个对话指向其他 act，最后一个 act（结局幕）的所有对话都指向自身（`nextActId` 指向该 act 自身即可，表示叙事结束）

### 3.5 总量要求

| 类型 | 最少 | 推荐 | 说明 |
|------|------|------|------|
| 主线幕 | 30 | 30-35 | 按时间线顺序推进 |
| 分支变体幕 | 10 | 15-25 | 由不同选择产生的替代剧情 |
| **总计** | **40** | **45-60** | 每个时代一个完整 JSON 文件 |

## 四、历史准确性规则（至关重要）

这是教育产品，历史准确性是生命线。以下规则**必须严格遵守**：

### 4.1 事件准确性
- 所有历史事件必须真实发生过，日期、地点、人物必须准确
- 虚构的对话和场景必须**不与已知史实矛盾**
- 如果某事件的准确年份有学术争议，采用主流教材（人教版统编教材）的说法

### 4.2 人物准确性
- 历史人物只能在他们在世的时期出现
- 人物的性格、立场、言语风格应基于历史记载
- 如果引用真实历史语录，必须在 `historicalNote` 中标注出处

### 4.3 时空一致性
- 人物不能在自己死亡后做决策
- 地理信息必须准确（城市、国家名称用当时的名称）
- 技术发明和物品不能出现在其被发明之前的时代

### 4.4 角色切换规则
- 如果玩家角色死亡，必须在 `sceneDescription` 中明确说明，并切换为新的玩家角色
- 切换时 `playerRole` 不变（在顶层结构中），但在场景描述中告知用户视角转换
- 示例：玩家扮演"亚历山大的将军"，亚历山大死后，场景描述为"亚历山大大帝在前323年病逝于巴比伦。作为他最信任的将领之一，你继承了帝国的部分领土......"

### 4.5 评分标准
- 90-100 分：符合历史最优决策，顺应历史大势，体现卓越的历史判断力
- 80-89 分：合理的决策，虽非最优但有历史依据
- 70-79 分：平庸或中性的决策，不会造成严重后果但错失了历史机遇
- 60-69 分：明显欠佳的决策，违背历史规律或导致不良后果

## 五、时代列表与详情

以下是全部 30 个世界史时代。每个时代包含 `eraId`、名称、时间范围、建议的玩家角色以及必须覆盖的核心内容。

### 5.1 九年级（Grade 9）-- 9 个时代

#### era 1: `w9-ancient-civilizations` -- 古代亚非文明
- **时间**：远古 - 公元前后
- **建议角色**：古埃及法老的顾问
- **核心内容**：尼罗河与古埃及文明（金字塔、法老制度）、两河流域文明（楔形文字、汉谟拉比法典、空中花园）、古印度文明（种姓制度、佛教创立）、古中国的早期成就
- **关键幕建议**：金字塔建造决策、汉谟拉比立法、佛教传播选择

#### era 2: `w9-ancient-europe` -- 古代欧洲文明
- **时间**：前800年 - 476年
- **建议角色**：雅典城邦的公民议员
- **核心内容**：古希腊城邦制度（雅典民主、斯巴达军事）、希腊古典文化（哲学、科学、艺术）、亚历山大东征、罗马共和国与帝国（凯撒、屋大维）、罗马法、西罗马帝国灭亡
- **关键幕建议**：雅典民主改革投票、亚历山大远征方向、凯撒渡过卢比孔河

#### era 3: `w9-medieval` -- 封建时代的欧洲与亚洲
- **时间**：476-1500年
- **建议角色**：查理曼宫廷的骑士
- **核心内容**：西欧封建制度（封君封臣制、庄园经济）、基督教会的权力、拜占庭帝国与查士丁尼法典、阿拉伯帝国与伊斯兰文明、日本大化改新与幕府制度
- **关键幕建议**：封地效忠仪式、十字军东征决策、日本大化改新

#### era 4: `w9-renaissance-exploration` -- 步入近代
- **时间**：14-16世纪
- **建议角色**：佛罗伦萨的人文主义学者

- **关键幕建议**：资助艺术家还是投资航海、哥伦布的航行方案、面对新大陆的决策

#### era 5: `w9-revolution` -- 资本主义制度的确立
- **时间**：17-18世纪
- **建议角色**：北美殖民地的独立运动领袖
- **核心内容**：英国资产阶级革命（议会斗争、光荣革命）、美国独立战争（波士顿倾茶事件、独立宣言、宪法）、法国大革命（攻占巴士底狱、人权宣言、拿破仑帝国）
- **关键幕建议**：光荣革命抉择、签署独立宣言、拿破仑称帝

#### era 6: `w9-industrial-revolution` -- 工业革命与马克思主义
- **时间**：18-19世纪
- **建议角色**：英国曼彻斯特的工厂主
- **核心内容**：第一次工业革命（瓦特蒸汽机、史蒂芬森火车、工厂制度）、工人阶级处境、马克思主义诞生（《共产党宣言》）、第一国际、巴黎公社
- **关键幕建议**：采用蒸汽机还是传统人力、面对工人罢工、巴黎公社

#### era 7: `w9-imperialism` -- 垄断资本主义与帝国主义
- **时间**：19世纪末-20世纪初
- **建议角色**：英国殖民地的总督
- **核心内容**：第二次工业革命（电力、内燃机、化学工业）、垄断组织形成、帝国主义瓜分世界、列强对非洲和亚洲的殖民、亚非拉民族独立运动（甘地、玻利瓦尔）
- **关键幕建议**：殖民扩张策略、面对当地抵抗运动、瓜分非洲会议

#### era 8: `w9-world-wars` -- 两次世界大战与冷战
- **时间**：1914-1991年
- **建议角色**：英国战时内阁的参谋官
- **核心内容**：第一次世界大战（萨拉热窝事件、凡尔赛条约）、俄国十月革命、第二次世界大战（诺曼底登陆、珍珠港、原子弹）、联合国成立、冷战（铁幕、柏林墙、古巴导弹危机）、殖民体系瓦解
- **关键幕建议**：一战后和约条款、慕尼黑协定抉择、诺曼底登陆决策、古巴导弹危机

#### era 9: `w9-contemporary` -- 当代世界
- **时间**：1991年至今
- **建议角色**：联合国维和行动的外交官
- **核心内容**：冷战结束与苏联解体、一超多强格局、经济全球化（WTO）、区域经济一体化（欧盟、NAFTA）、互联网与科技革命、可持续发展
- **关键幕建议**：苏联解体时的外交应对、全球化与本地经济、气候变化应对

### 5.2 高中（Senior World）-- 10 个时代

## 九、技术集成说明

生成的 JSON 文件将放置在项目的 `src/data/narratives/` 目录下，并通过以下机制加载：

1. 文件命名：`{eraId}.json`，如 `w9-ancient-civilizations.json`
2. 加载方式：前端通过 `fetch` API 从 `/data/narratives/` 路径异步加载
3. 数据校验：TypeScript 接口定义在 `src/lib/data.ts` 中（`NarrativeData`、`NarrativeAct`、`NarrativeDialogue`、`NpcDialogue`）
4. 文件映射：需要在 `src/lib/data.ts` 的 `NARRATIVE_FILES` 对象中添加 `eraId` 到文件名的映射
5. 对话展示：系统从每幕的 9 个对话中随机抽取 3 个展示给用户
6. 分支逻辑：选择对话后，系统读取 `nextActId` 字段来确定下一个要展示的幕
7. 评分系统：用户每选择一个对话获得对应的 score，最终计算平均分并给予等级评定

### 当前 NarrativeData TypeScript 接口参考

```typescript
export interface NarrativeData {

  eraName: string;
  playerRole: string;
  acts: NarrativeAct[];
}

export interface NarrativeAct {
  id: string;
  title: string;
  isKeyAct: boolean;
  sceneDescription: string;
  npcDialogues: NpcDialogue[];
  dialogues: NarrativeDialogue[];
}

export interface NpcDialogue {
  speaker: string;
  text: string;
  avatar: string;
}

export interface NarrativeDialogue {
  id: string;
  text: string;
  speaker: string;
  avatar: string;
  score: number;
  triggeredEvent: string;
  historicalNote: string;
  consequence: string;
  // nextActId 字段将在前端分支逻辑中使用
  nextActId: string;
}

export interface PlayerChoice {
  actId: string;
  dialogueId: string;
  score: number;
  triggeredEvent: string;
}
```

> **注意**：当前代码中 `NarrativeDialogue` 接口尚未包含 `nextActId` 字段。生成数据后，需要同步更新此接口定义，并修改沉浸式页面的导航逻辑（当前是简单的 `actIndex + 1` 顺序推进），改为根据选择的 `nextActId` 查找对应 act 并跳转。
